# TwitterClone
ポートフォリオとして作成した「twitterのクローンアプリ」です。
「新規登録」
「ログイン」
「投稿」（画像およびテキスト）
「プロフィール編集」
「ログアウト」
の基本機能が使えます。

# 開発環境

言語
- PHP 7.3.7

DBMS 
- MySQL 5.5.62

ストレージ（画像保存）
- Amazon Web Services S3

CSSフレームワーク
- Uikit

端末OS
- Windows10 64bit

仮想環境
- Vagrant 2.2.5
- VirtualBox 5.1.30
- CentOS 6.10


# 開発目的

入門教材における基礎学習が終わり、実戦形式での開発をすることで基礎知識への理解を深める。
フレームワークを学習する前に、実際の処理の流れを確認する。


# 反省点

コードの重複
- 同じような処理が別のファイルに記述されている部分が多い。
- オブジェクト指向への理解が浅く、各ページとリンクするクラスを作っているだけなので
　ほぼ手続き型のような記述になり冗長になっている。
 

設計の甘さ
- 基礎設計を紙に書き出してはいたが、作成中に軌道修正が多々あった。
　必要な機能、欲しい機能の選定が不十分だった。
- 自身の経験不足により「できること」と「必要な時間」の計算が出来ず、
　臨機応変に作る形となってしまった事が原因と思われる。
 
 
未完成
- 基本機能は完成しているが、当初予定していた機能が実装しきれていない。
- 今後、少しづつ手を加える予定。


# まとめ

基礎教材レベルでは触れる技術が限られているため、自身での開発をすることで
新たな技術に触れる機会を広げられると感じた。

例えば今回の開発では
「アプリからheroku上に画像をアップロードしても、前回プッシュ時の状態にリセットされてしまう」
ということが分かり、クラウドストレージに画像を保存してやりとりする必要が出てきた。
そこで、AWSを使用することに行きつき、新たにクラウドストレージとのファイルのやり取りを学ぶことができた。

今回は時間的な都合もあり、「自身のレベルで出来るのか」ということを考えてしまったため
結果的に上記ような機会を狭めてしまったが、それでも学びは多かった。

時間的制約がなければ、という話になるが
「学習」という点においては「こんなサービスを作りたい」ということにフォーカスを当て
「自分ありき」ではなく「サービスありき」で開発をし、必要な技術をどんどん吸収する形の方が良いと感じた。
今後はそうした機会を増やしていきたい。
